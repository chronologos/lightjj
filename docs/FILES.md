# File Guide

Per-file map of the codebase: what each file does plus the invariants, gotchas, and design notes that matter when changing it. The compact one-line-per-file index lives in [CLAUDE.md](../CLAUDE.md#project-structure); this is the detailed version. Read a file's entry before modifying the file or adding a new caller of its exports; update the entry when adding, removing, or renaming a file or when its contract changes (and keep the CLAUDE.md one-liner in sync). Test files get entries only when the test setup itself carries invariants worth noting.

See [ARCHITECTURE.md](ARCHITECTURE.md) for the system-level design these files implement.

## cmd/lightjj/

- **`main.go`** ŌĆö CLI entry point, flag parsing, embeds frontend-dist/
- **`frontend_embed.go`** ŌĆö `-tags embed` build: frontendHandler serves the go:embed'd frontend-dist/.
- **`frontend_stub.go`** ŌĆö Default (no embed tag) build: frontendHandler returns a static "frontend not bundled" help page pointing at release binaries or a `-tags embed` source build, so plain `go install` binaries don't 404.
- **`session_file.go`** ŌĆö Agent port-discovery: writes {pid,addr,repo_dir,mode} to `$XDG_RUNTIME_DIR/lightjj/sessions/<pid>.json` after bind; stale-pid sweep; symlink-race-hardened TempDir fallback (verifyOwnedDir double-check). No-op on Windows. Agents match by repo_dir instead of needing the random port.
- **`session_file_unix.go`** ŌĆö verifyOwner: checks the session dir is owned by the current uid (unix builds).
- **`session_file_other.go`** ŌĆö verifyOwner no-op stub for non-unix builds (Windows short-circuits earlier in writeSessionFile).
- **`api_cmd.go`** ŌĆö `lightjj api METHOD PATH [BODY]` + `lightjj sessions` subcommands: loopback-only HTTP client (validateAddr rejects non-localhost) with session-file discovery, for agent harnesses whose sandboxes denylist curl/wget. Security model + discovery algorithm in [design-notes/api-cli.md](design-notes/api-cli.md).
- **`skill_cmd.go`** ŌĆö `lightjj skill [install]`: prints or installs the go:embed'd SKILL.md (the agent-facing guide to `lightjj api`); install refuses to overwrite a file that doesn't carry the lightjj marker unless `--force`.
- **`apply_hunks.go`** ŌĆö `--apply-hunks=<spec.json>` re-entry path for `jj split --tool lightjj-hunks`: dumb writer applying the frontend-synthesized write/revert/delete spec into jj's $right dir, with path-escape checks. All hunk-patching logic lives in frontend hunk-apply.ts.

## internal/runner/

CommandRunner interface + implementations

- **`runner.go`** ŌĆö Interface definition (Run, RunWithInput, RunForMutation, StreamCombined, RunRaw, WriteFile)
- **`local.go`** ŌĆö LocalRunner: exec("jj", args) with configurable Binary. WriteFile does symlink-escape hardening (EvalSymlinks on parent + Lstat leaf) before os.WriteFile. ResolveWorkspaceRoot rejects forgotten workspaces: `jj workspace root` exits 0 inside a forgotten workspace's dir, so a `jj log -r @` probe (probeWorkspace) follows it ŌĆö tab create 400s and the startup restore loop log-and-skips instead of mounting a permanently broken tab.
- **`ssh.go`** ŌĆö SSHRunner: wraps jj args in ssh command. WriteFile pipes content via `cd <repo> && cat > <path>` over stdin ŌĆö no symlink check (same trust boundary as remote shell).

## internal/api/

HTTP handlers

- **`server.go`** ŌĆö Route registration, runMutation, op-id caching, workspace store reader, helpers. For pure decodeŌåÆvalidateŌåÆbuild mutations the route line IS the handler (`reg("POST /api/new", mutation(s, newRequest.validate, newRequest.build))`); the endpoint table in docs/ARCHITECTURE.md is drift-checked by architecture_doc_test.go. `Server.OpenTabRoots` (func field, injected by TabManager.addLocked; nil in tests/standalone) reports all open tab roots for cross-tab guards.
- **`handlers.go`** ŌĆö Endpoint implementations, flag validation, and `mutation[Req](s, validate, build)` ŌĆö the generic decodeŌåÆvalidateŌåÆbuildŌåÆrunMutation factory. Pure mutations exist as request structs with `validate()`/`build()` method pairs (no named handler func); `opMutation`/`bookmarkMutation`/`bookmarkRevMutation`/`bookmarkRemoteMutation` are thin adapters over `mutation()` for families sharing one request shape. Hand-rolled handlers are the ones with extra behavior: stdin (describe), streaming (git push/fetch/alias/index-paths), runner-backed validation (alias), tempfiles (split-hunks, merge-resolve), non-400 guards (workspace forget), server-state builds (workspace add), post-mutation side effects (snapshot, file-write, unlock-repo). Every success response uses the MutationResult envelope `{output, warnings?}` (apiVersion 2 unified the legacy `{ok}`/`{status}` shapes). Workspace name validation is two-tier: `workspaceNameRe` (strict, ADD only ŌĆö name becomes a dir suffix) vs `validateWorkspaceRefName` (loose, forget/rename ŌĆö jj allows dots/slashes in existing names; flag injection blocked by the `--` in the command builders). `workspaceNameMutation(build, guard)` is the forget/rename factory (decodeŌåÆvalidateŌåÆguardŌåÆrunMutation; stays hand-rolled because guard returns non-400 statuses); `forgetOpenTabGuard` 409s when the target workspace's path (via `workspacePathMap`, shared with handleWorkspaces) matches any open tab root, and allows when the path can't be resolved.
- **`handlers_test.go`** ŌĆö Handler tests with MockRunner
- **`architecture_doc_test.go`** ŌĆö Drift guard for the docs/ARCHITECTURE.md endpoint table: parses `| METHOD | \`/api/...\` |` rows and asserts two-way sync with `routes()` (every registered route documented, every documented `/api` row registered). Tab routes (`/tabs`, `/tab/{id}/*`) are exempt ŌĆö they're TabManager's, not `routes()`'. Sibling of agent_docs_test.go (the agent_api.md guard).
- **`integration_test.go`** ŌĆö Integration tests (build-tagged)
- **`watcher.go`** ŌĆö fsnotify on .jj/repo/op_heads/heads/ + SSE push, periodic util snapshot. SSH mode uses `sshPollLoop` (`PollOpId` ŌĆö implicit snapshot + op-id in one round trip; no remote deps; CAS against pre-poll cachedOp so a concurrent runMutation advance isn't regressed). `interval <= 0` gates both loops (parity). Stale-WC detection: both loops check `isStaleWCError()` ŌåÆ `setStale(bool)` (staleMu-serialized Swap+broadcast ŌĆö handler clear can't race a loop set and emit sentinels out-of-order; Swap edges don't re-fire so the race has no self-heal) ŌåÆ `evStaleWC`/`evFreshWC` sentinels ("!"-prefixed on the existing `chan string`) ŌåÆ frontend non-dismissable warning with "Update stale" action. Both handleSnapshot + handleWorkspaceUpdateStale call `clearStale()`. `handleEvents` refreshes cachedOp if empty (read-only session path), emits current stale state unconditionally on connect (both branches ŌĆö false branch covers SSE-dropped-during-staleness ŌåÆ CLI-recovered ŌåÆ reconnect), per-write WriteDeadline extension (not blanket disable ŌĆö dead TCP surfaces in Ōēż85s).
- **`watcher_test.go`** ŌĆö Broadcaster tests (subscribe/broadcast/drop), sshPollLoop tests via seqRunner (hook field for mid-call injection), setStale edge-only invariant, handleEvents refresh-if-empty table, CAS-guards-concurrent-advance
- **`tabs.go`** ŌĆö TabManager: per-tab Server + Watcher mounted at /tab/{id}/; TabResolve + TabFactory injection (mode-agnostic validation via closure); openTabs persisted to config.json (tab 0 excluded ŌĆö fights with -R on next launch); startup restore re-runs resolve() so moved repos log-and-skip. `OpenTabRoots()` (RLock) returns every open tab's root; addLocked injects it into each mounted Server (feeds the workspace-forget cross-tab guard).
- **`config.go`** ŌĆö Server-side JSONC config (tailscale/hujson); mergeAndWriteConfig applies per-key RFC-6902 add patches so comments on existing keys survive; fresh installs (ENOENT) seed from configTemplate, corrupt files NEVER reseed ŌĆö 422 + frontend warning keeps user's bad file untouched (ErrConfigUnparseable sentinel). writePersistedTabs filter-merges openTabs through the same patch path. GET /api/config returns Standardized JSON (422 on corrupt); GET/POST /api/config/raw round-trips raw JSONC for ConfigModal (text/plain only, 1MB cap, isLocalOrigin guard).
- **`config_jsonc.go`** ŌĆö hujson helpers: standardizeJSONC (bytes.Clone first ŌĆö hujson.Standardize mutates input via aliased Extra slices), unmarshalJSONC, patchConfigKeys. The centralised "Standardize before Unmarshal" idiom ŌĆö any new json.Unmarshal on config bytes must go through unmarshalJSONC or break the first time the user adds a comment.
- **`config_template.go`** ŌĆö First-run JSONC template string (in-tree constant, not a file). Teaching comments over theme, fontSize, editorArgs, font families. Kept in sync with frontend defaults and [CONFIG.md](CONFIG.md) by convention ŌĆö see the top-of-file comment.
- **`jsonstore.go`** ŌĆö Generic `jsonCollection[T hasID]` keyed flat-file JSON-array store (key ŌåÆ one file via `pathFor`, items addressed by ID). Owns the machinery annotations + doc-comments previously hand-mirrored: per-collection read-modify-write mutex, merge-on-upsert (preserve-on-empty, runs BEFORE stamp so merge detects "client omitted X" by zero value), server-side `stamp` defaults, configurable `deleteMatch` (cascade), delete-file-when-last-item-removed, and `UpsertBatch` (whole batch under one lock). Collections MUST be package-level vars, not Server fields ŌĆö store files are shared across tabs, so the mutex must be too. Errors are `storeError{status,msg}`; handlers map via `storeErrStatus` (bad key ŌåÆ 400, I/O ŌåÆ 500). Low-level readJSONStore/atomicWriteJSON also live here.
- **`annotations.go`** ŌĆö Per-changeId review comments (annotations/{changeId}.json): `Annotation` type + `annotationStore` collection config (pathFor=annotationsPath with changeId regex validation; merge preserves resolution/resolvedAtCommitId AND legacy Status under the same condition, plus createdAt; stamp fills createdAt) + thin GET/POST/DELETE handlers. DELETE without id = clear-all (DeleteAll).
- **`doc_comments.go`** ŌĆö Doc-mode range-anchored comments/suggestions: `DocComment`/`DocAnchor` types + `docCommentStore` collection config (key = RepoPath+"|"+cleanDocPath(filePath) via `docCommentKey`, filename = sha256(key)[:16]; merge preserves resolution/resolvedAt/createdAt; stamp fills randID + createdAt; deleteMatch cascades to replies via ParentId) + GET/POST/DELETE/batch handlers. cleanDocPath: ToSlash+Clean+strip ./, reject ../ and / (400). POST validates anchor.selection. DELETE requires id (no clear-all). Batch validates ALL comments before writing ANY (all-or-nothing via UpsertBatch).
- **`agent_docs.go`** ŌĆö `GET /api/agent` serves go:embed agent_api.md (the agent-facing contract). agent_docs_test.go fetches it and asserts every /api/* path mentioned is a registered route ŌĆö the structural guard against doc/route drift.
- **`symbol.go`** ŌĆö rg-backed go-to-definition for `GET /api/symbol` (Tier 1 ŌĆö heuristic, no language server): per-language definition regex run against the working copy via `rg --json` through RunRaw (so it works in SSH mode), identifier-gated `name` param, capped hits with leading doc-comment context.
- **`focus.go`** ŌĆö `GET/POST /api/focus`: frontend-reported "what is the user looking at" (change_id/commit_id/active view/doc path) for the agent steering loop. Server-stamped UpdatedAt, per-tab singleton (each tab is its own Server), field-length caps.
- **`open.go`** ŌĆö Open-in-$EDITOR: buildEditorArgv ({file}/{line} substitution, argv[0] validation), handleOpenFile. Setsid via build-tagged detachProcess (open_unix.go / open_windows.go)
- **`open_unix.go`** ŌĆö detachProcess via Setsid (!windows) so Ctrl+C on lightjj doesn't kill the spawned editor.
- **`open_windows.go`** ŌĆö detachProcess no-op (windows builds).
- **`gzip.go`** ŌĆö Gzip response middleware (lazy-init writer, Flush passthrough for SSE)
