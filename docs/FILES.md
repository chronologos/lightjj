# File Guide

Per-file map of the codebase: what each file does plus the invariants, gotchas, and design notes that matter when changing it. The compact one-line-per-file index lives in [CLAUDE.md](../CLAUDE.md#project-structure); this is the detailed version. Read a file's entry before modifying the file or adding a new caller of its exports; update the entry when adding, removing, or renaming a file or when its contract changes (and keep the CLAUDE.md one-liner in sync). Test files get entries only when the test setup itself carries invariants worth noting.

See [ARCHITECTURE.md](ARCHITECTURE.md) for the system-level design these files implement.

## cmd/lightjj/

- **`main.go`** ÔÇö CLI entry point, flag parsing, embeds frontend-dist/. Startup order matters for persistence: MigrateConfigIfNeeded Ôćĺ MigrateStateIfNeeded Ôćĺ tab restore (ReadPersistedTabs) ÔÇö the state migration must run before tab restore or upgraded users lose their tabs on first launch.
- **`frontend_embed.go`** ÔÇö `-tags embed` build: frontendHandler serves the go:embed'd frontend-dist/.
- **`frontend_stub.go`** ÔÇö Default (no embed tag) build: frontendHandler returns a static "frontend not bundled" help page pointing at release binaries or a `-tags embed` source build, so plain `go install` binaries don't 404.
- **`session_file.go`** ÔÇö Agent port-discovery: writes {pid,addr,repo_dir,mode} to `$XDG_RUNTIME_DIR/lightjj/sessions/<pid>.json` after bind; stale-pid sweep; symlink-race-hardened TempDir fallback (verifyOwnedDir double-check). No-op on Windows. Agents match by repo_dir instead of needing the random port.
- **`session_file_unix.go`** ÔÇö verifyOwner: checks the session dir is owned by the current uid (unix builds).
- **`session_file_other.go`** ÔÇö verifyOwner no-op stub for non-unix builds (Windows short-circuits earlier in writeSessionFile).
- **`api_cmd.go`** ÔÇö `lightjj api METHOD PATH [BODY]` + `lightjj sessions` subcommands: loopback-only HTTP client (validateAddr rejects non-localhost) with session-file discovery, for agent harnesses whose sandboxes denylist curl/wget. Security model + discovery algorithm in [design-notes/api-cli.md](design-notes/api-cli.md).
- **`skill_cmd.go`** ÔÇö `lightjj skill [install]`: prints or installs the go:embed'd SKILL.md (the agent-facing guide to `lightjj api`); install refuses to overwrite a file that doesn't carry the lightjj marker unless `--force`.
- **`apply_hunks.go`** ÔÇö `--apply-hunks=<spec.json>` re-entry path for `jj split --tool lightjj-hunks`: dumb writer applying the frontend-synthesized write/revert/delete spec into jj's $right dir, with path-escape checks. All hunk-patching logic lives in frontend hunk-apply.ts.

## internal/jj/

Command builders + data models (PURE ÔÇö no I/O, no side effects)

- **`commands.go`** ÔÇö Functions that return []string args for jj subcommands, plus the parsers for their template output. Two template/parser styles coexist: legacy delimiter-positional (`\x1F`/`\x1E`/`\x1D` + SplitN ÔÇö LogGraph, OpLog, FilesBatch, ConflictList, ÔÇŽ) and json()-composed (one JSON object per line, `encoding/json` parse ÔÇö `OpLogJSON`/`ParseOpLogJSON`, the reference migration). NEW parsers must use the json() style (jj Ôëą 0.31, `JSONTemplates` gate ÔÇö see CLAUDE.md Go conventions); existing delimiter parsers migrate opportunistically.
- **`commit.go`** ÔÇö Commit model with ChangePrefix/CommitPrefix, Immutable, Divergent, WorkingCopies, Mine, AuthorEmail, Timestamp
- **`bookmark.go`** ÔÇö Bookmark model + output parsers; ParseBookmarkListOutput/ParseRemoteListOutput take defaultRemote param for sort order. `BookmarkRemote` carries per-ref `AuthorName`/`AuthorEmail` (tip-commit author ÔÇö display + author-filter target), `CommitTs` (committer timestamp, epoch SECONDS ÔÇö the "Recent" sort key; frontend derives the relative age from it), and `Mine` (`normal_target.mine()` ÔÇö author is me; the UI suppresses the author label on your own branches). Template is 14 \x1F-fields (bookmarkListTemplate in commands.go); parser guards `len(parts) < 14`. No jj-version gate ÔÇö author()/committer().timestamp().format("%s")/mine() are core template builtins.
- **`alias.go`** ÔÇö jj config alias parser. Handles both the simple array form (`aliases.l = [...]`) and the table form (`aliases.s.definition = [...]` + `aliases.s.doc = "..."`, jj Ôëą 0.42's documented `{ definition, doc }` shape) ÔÇö sub-keys coalesce by name into one `Alias{Name, Command, Doc}`; entries that never yield a command are dropped (a stray `.doc` is ignored). `Doc` flows to the Command Palette hint. No jj-version gate ÔÇö `jj config list` emits this format identically across versions; it's just config the user may or may not have written.
- **`file_change.go`** ÔÇö FileChange model, FilesTemplate (single-call file stats + conflict info), ParseFilesTemplate
- **`divergence.go`** ÔÇö DivergenceEntry, Divergence() template builder, ParseDivergence. committer_ts DELIBERATELY absent (structurally inverted for --at-op). splitNonEmpty helper.
- **`selected_revisions.go`** ÔÇö Multi-revision selection helper
- **`version.go`** ÔÇö Semver type + ParseSemver + named feature gates (WorkspaceRootTmpl, ChangedPathIndex) + `FeatureGates` wire registry (gate name Ôćĺ minimum Semver). GET /api/info resolves every registry entry through `Server.jjSupports` (pessimistic) and ships the booleans as `features` ÔÇö the frontend reads those instead of keeping a version table, so this map is the single authority on minimum versions. Keys are a wire contract with frontend `jjSupports(name)`.
- **`workspace_store.go`** ÔÇö Protobuf parser for .jj/repo/workspace_store/index (nameÔćĺpath map). <0.40 fallback ÔÇö Ôëą0.40 uses WorkspaceRef.root() in the WorkspaceList template.
- **`workspace_store_test.go`** ÔÇö Parser tests with real binary data

## internal/runner/

CommandRunner interface + implementations

- **`runner.go`** ÔÇö Interface definition (Run, RunWithInput, RunForMutation, StreamCombined, RunRaw, WriteFile)
- **`local.go`** ÔÇö LocalRunner: exec("jj", args) with configurable Binary. WriteFile does symlink-escape hardening (EvalSymlinks on parent + Lstat leaf) before os.WriteFile. ResolveWorkspaceRoot rejects forgotten workspaces: `jj workspace root` exits 0 inside a forgotten workspace's dir, so a `jj log -r @` probe (probeWorkspace) follows it ÔÇö tab create 400s and the startup restore loop log-and-skips instead of mounting a permanently broken tab.
- **`ssh.go`** ÔÇö SSHRunner: wraps jj args in ssh command. WriteFile pipes content via `cd <repo> && cat > <path>` over stdin ÔÇö no symlink check (same trust boundary as remote shell).

## internal/api/

HTTP handlers

- **`server.go`** ÔÇö Route registration, runMutation, op-id caching, workspace store reader, helpers. For pure decodeÔćĺvalidateÔćĺbuild mutations the route line IS the handler (`reg("POST /api/new", mutation(s, newRequest.validate, newRequest.build))`); the endpoint table in docs/ARCHITECTURE.md is drift-checked by architecture_doc_test.go. `Server.OpenTabRoots` (func field, injected by TabManager.addLocked; nil in tests/standalone) reports all open tab roots for cross-tab guards. The op-id cache is encapsulated behind `getOpId()`/`setOpId()`/`casOpId()` ÔÇö the `cachedOp`/`cachedMu` fields are an implementation detail nothing else (besides `refreshOpId`, which composes them with the actual jj read) should touch; the SSH poll loop's compare-and-swap goes through `casOpId`, never the mutex directly. `trySnapshot()` is the pause-respecting snapshot path used by snapshotLoop and handleFileWrite; handleSnapshot checks the pause inline instead because it needs runMutation's response envelope and runMutation itself bumps the pause counter.
- **`handlers.go`** ÔÇö Endpoint implementations, flag validation, and `mutation[Req](s, validate, build)` ÔÇö the generic decodeÔćĺvalidateÔćĺbuildÔćĺrunMutation factory. Pure mutations exist as request structs with `validate()`/`build()` method pairs (no named handler func); `opMutation`/`bookmarkMutation`/`bookmarkRevMutation`/`bookmarkRemoteMutation` are thin adapters over `mutation()` for families sharing one request shape. Hand-rolled handlers are the ones with extra behavior: stdin (describe), streaming (git push/fetch/alias/index-paths), runner-backed validation (alias), tempfiles (split-hunks, merge-resolve), non-400 guards (workspace forget), server-state builds (workspace add), description reads (squash `source`/`combine` modes read the involved descriptions via `composeSquashMessage` Ôćĺ `-m`, issue #22), post-mutation side effects (snapshot, file-write, unlock-repo). Every success response uses the MutationResult envelope `{output, warnings?}` (apiVersion 2 unified the legacy `{ok}`/`{status}` shapes). Workspace name validation is two-tier: `workspaceNameRe` (strict, ADD only ÔÇö name becomes a dir suffix) vs `validateWorkspaceRefName` (loose, forget/rename ÔÇö jj allows dots/slashes in existing names; flag injection blocked by the `--` in the command builders). `workspaceNameMutation(build, guard)` is the forget/rename factory (decodeÔćĺvalidateÔćĺguardÔćĺrunMutation; stays hand-rolled because guard returns non-400 statuses); `forgetOpenTabGuard` 409s when the target workspace's path (via `workspacePathMap`, shared with handleWorkspaces) matches any open tab root, and allows when the path can't be resolved. `handleWorkspaceUpdateStaleOther` (issue #21) recovers a SIBLING workspace by name: resolve its root via `workspacePathMap`, then run `jj -R <path> workspace update-stale` through **RunRaw** (NOT runMutation) ÔÇö the runner injects the tab's own `-R RepoDir` and jj rejects two `--repository` flags, so RunRaw (which adds no `-R`) is the only way to target a sibling in both modes; path comes from jj's own output so it's never request-injected; 422 when the path is unknown (predates jj's workspace_store index). The current workspace keeps its watcher-cleared `handleWorkspaceUpdateStale`.
- **`handlers_test.go`** ÔÇö Handler tests with MockRunner
- **`architecture_doc_test.go`** ÔÇö Drift guard for the docs/ARCHITECTURE.md endpoint table: parses `| METHOD | \`/api/...\` |` rows and asserts two-way sync with `routes()` (every registered route documented, every documented `/api` row registered). Tab routes (`/tabs`, `/tab/{id}/*`) are exempt ÔÇö they're TabManager's, not `routes()`'. Sibling of agent_docs_test.go (the agent_api.md guard).
- **`integration_test.go`** ÔÇö Integration tests (build-tagged)
- **`watcher.go`** ÔÇö fsnotify on .jj/repo/op_heads/heads/ + SSE push, periodic util snapshot. Internals are typed: broadcasts are `sseEvent{name, data, droppable}` values (constructors `opEvent`/`navEvent`/`pollFailEvent`, sentinel vars `evStaleWC`/`evFreshWC`/`evPollOk`) that map 1:1 onto the `event:`/`data:` wire frames ÔÇö the names/payload shapes are the api.ts contract and must not change; `handleEvents` is a single formatted write with zero event-type dispatch. `broadcast()` drops `droppable` events (op-ids, nav hints) when a subscriber's buffer is full, but EVICTS the oldest queued event to deliver non-droppable sentinels (stale-wc/fresh-wc, pollfail/pollok) ÔÇö they're edge-triggered and never re-sent, so a dropped one would stick the client until reconnect. Both background loops classify probe results through ONE shared `probeTracker` (`ok`/`staleWC`/`fail`): it owns the consecutive-error counter, the 3rd-then-every-12th log throttle, the `pollFailThreshold` UI gate, and the unconditional poll-fail-clear discipline (the stale-WC-interlude-leaves-pollFailed-stuck bug previously had to be fixed once per loop ÔÇö the invariant comments now live once, on the struct). SSH mode uses `sshPollLoop` (`PollOpId` ÔÇö implicit snapshot + op-id in one round trip; no remote deps; `Server.casOpId` CAS against the pre-poll cache value so a concurrent runMutation advance isn't regressed). `interval <= 0` disables both loops (parity). Stale-WC detection: `isStaleWCError()` Ôćĺ `setStale(bool)` (staleMu-serialized Swap+broadcast ÔÇö handler clear can't race a loop set and emit sentinels out-of-order; Swap edges don't re-fire so the race has no self-heal) Ôćĺ `evStaleWC`/`evFreshWC` Ôćĺ frontend non-dismissable warning with "Update stale" action. Both handleSnapshot + handleWorkspaceUpdateStale call `clearStale()`. Snapshot-pause counters are shared per canonical repo path via the package-level `pauseCounterFor` registry, resolved at watcher CONSTRUCTION (`pauseKey`: RepoDir locally, RepoPath over SSH) ÔÇö same-path tabs share one counter with no post-construction field swapping. `handleEvents` refreshes the op-id if the cache is empty (read-only session path), emits current stale + poll-fail state unconditionally on connect (both branches ÔÇö false branch covers SSE-dropped-during-staleness Ôćĺ CLI-recovered Ôćĺ reconnect), per-write WriteDeadline extension (not blanket disable ÔÇö dead TCP surfaces in ÔëĄ85s).
- **`watcher_test.go`** ÔÇö Broadcaster tests (subscribe/broadcast, droppable-drop on full buffer, sentinel eviction on full buffer), sshPollLoop tests via seqRunner (hook field for mid-call injection), setStale edge-only invariant, handleEvents refresh-if-empty table, CAS-guards-concurrent-advance, stale-interlude-clears-pollfail on BOTH loops (the shared probeTracker discipline), shared-pause-counter-by-path
- **`tabs.go`** ÔÇö TabManager: per-tab Server + Watcher mounted at /tab/{id}/; TabResolve + TabFactory injection (mode-agnostic validation via closure); openTabs persisted to **state.json** via `SetOpenTabs` (state.go ÔÇö NOT config.json, so tab churn never runs the JSONC patch machinery; tab 0 excluded ÔÇö fights with -R on next launch); startup restore re-runs resolve() so moved repos log-and-skip. `OpenTabRoots()` (RLock) returns every open tab's root; addLocked injects it into each mounted Server (feeds the workspace-forget cross-tab guard). NewTabManager also registers the host-scoped config + state routes on its Mux. addLocked wires only the SSE subscriber hooks (idle shutdown); snapshot-pause sharing across same-path tabs is NOT wired here ÔÇö watchers resolve their shared counter at construction (`pauseCounterFor`, watcher.go).
- **`config.go`** ÔÇö Server-side JSONC config (tailscale/hujson) for HUMAN-edited keys only; mergeAndWriteConfig applies per-key RFC-6902 add patches so comments on existing keys survive; fresh installs (ENOENT) seed from configTemplate, corrupt files NEVER reseed ÔÇö 422 + frontend warning keeps user's bad file untouched (ErrConfigUnparseable sentinel). Machine-written keys (openTabs, recentActions) live in state.json (state.go), never here. GET /api/config returns Standardized JSON (422 on corrupt); GET/POST /api/config/raw round-trips raw JSONC for ConfigModal (text/plain only, 1MB cap, isLocalOrigin guard). Atomic writes go through the shared `atomicWriteFile` (jsonstore.go).
- **`config_jsonc.go`** ÔÇö hujson helpers: standardizeJSONC (bytes.Clone first ÔÇö hujson.Standardize mutates input via aliased Extra slices), unmarshalJSONC, patchConfigKeys, removeConfigKeys (RFC-6902 "remove"; only pass keys that exist ÔÇö removing an absent path errors). The centralised "Standardize before Unmarshal" idiom ÔÇö any new json.Unmarshal on config bytes must go through unmarshalJSONC or break the first time the user adds a comment.
- **`config_template.go`** ÔÇö First-run JSONC template string (in-tree constant, not a file). Teaching comments over theme, fontSize, editorArgs, font families. Kept in sync with frontend defaults and [CONFIG.md](CONFIG.md) by convention ÔÇö see the top-of-file comment.
- **`state.go`** ÔÇö Machine-state store: state.json (plain JSON, NOT JSONC) next to config.json, holding values lightjj writes itself ÔÇö `openTabs` (PersistedTab; SetOpenTabs filter-merges by (mode,host) so concurrent sessions don't stomp each other; ReadPersistedTabs for startup restore) and `recentActions` (namespace Ôćĺ key Ôćĺ ms timestamp; SetRecentActions whole-section replace). One `stateMu` over every read-modify-write; atomic writes via `atomicWriteFile`. Missing/zero-byte/corrupt files all read as fresh (machine state is recoverable; nothing to protect with a 422). `MigrateStateIfNeeded` (main.go startup, after MigrateConfigIfNeeded, before tab restore) imports legacy openTabs/recentActions from config.json then removes them via removeConfigKeys ÔÇö import is written BEFORE removal so failure can't lose data. HTTP: GET/POST /api/state/recent-actions (package-level handlers, host-scoped like config; POST has the isLocalOrigin guard).
- **`jsonstore.go`** ÔÇö Generic `jsonCollection[T hasID]` keyed flat-file JSON-array store (key Ôćĺ one file via `pathFor`, items addressed by ID). Owns the machinery annotations + doc-comments previously hand-mirrored: per-collection read-modify-write mutex, merge-on-upsert (preserve-on-empty, runs BEFORE stamp so merge detects "client omitted X" by zero value), server-side `stamp` defaults, configurable `deleteMatch` (cascade), delete-file-when-last-item-removed, and `UpsertBatch` (whole batch under one lock). Collections MUST be package-level vars, not Server fields ÔÇö store files are shared across tabs, so the mutex must be too. Errors are `storeError{status,msg}`; handlers map via `storeErrStatus` (bad key Ôćĺ 400, I/O Ôćĺ 500). Low-level readJSONStore/atomicWriteJSON live here too, plus `atomicWriteFile` ÔÇö the SINGLE temp-file+rename primitive shared by jsonstore/config/state (it prevents torn writes only; lost-update prevention is the caller's mutex).
- **`annotations.go`** ÔÇö Per-changeId review comments (annotations/{changeId}.json): `Annotation` type + `annotationStore` collection config (pathFor=annotationsPath with changeId regex validation; merge preserves resolution/resolvedAtCommitId AND legacy Status under the same condition, plus createdAt; stamp fills createdAt) + thin GET/POST/DELETE handlers. DELETE without id = clear-all (DeleteAll).
- **`doc_comments.go`** ÔÇö Doc-mode range-anchored comments/suggestions: `DocComment`/`DocAnchor` types + `docCommentStore` collection config (key = RepoPath+"|"+cleanDocPath(filePath) via `docCommentKey`, filename = sha256(key)[:16]; merge preserves resolution/resolvedAt/createdAt; stamp fills randID + createdAt; deleteMatch cascades to replies via ParentId) + GET/POST/DELETE/batch handlers. cleanDocPath: ToSlash+Clean+strip ./, reject ../ and / (400). POST validates anchor.selection. DELETE requires id (no clear-all). Batch validates ALL comments before writing ANY (all-or-nothing via UpsertBatch).
- **`agent_docs.go`** ÔÇö `GET /api/agent` serves go:embed agent_api.md (the agent-facing contract). agent_docs_test.go fetches it and asserts every /api/* path mentioned is a registered route ÔÇö the structural guard against doc/route drift.
- **`symbol.go`** ÔÇö rg-backed go-to-definition for `GET /api/symbol` (Tier 1 ÔÇö heuristic, no language server): per-language definition regex run against the working copy via `rg --json` through RunRaw (so it works in SSH mode), identifier-gated `name` param, capped hits with leading doc-comment context.

- **`open.go`** ÔÇö Open-in-$EDITOR: buildEditorArgv ({file}/{line} substitution, argv[0] validation), handleOpenFile. Setsid via build-tagged detachProcess (open_unix.go / open_windows.go). ## frontend/

Svelte 5 SPA (Vite + TypeScript + pnpm)

- **`src/AppShell.svelte`** ÔÇö Tab-switch host. Palette commands: the `!inlineMode` gate is injected per-CATEGORY where `commands` is assembled (`INLINE_GATED_CATEGORIES` ÔÇö Revisions/Git/Bookmarks/Workspace/Annotations/Aliases); per-entry `when()` only adds further conditions, and entries in non-gated categories (Navigation/View/Help/Actions) spell their own gate where needed. ÔÇŽ :` chains. Workspace state: `currentWorkspace === ''` means UNKNOWN (initial value; a failed workspaces fetch keeps it ÔÇö keepValueOnError) ÔÇö `openWorkspaceContextMenu` fails safe on unknown (non-destructive items only, no Forget/Rename). **Selection is identity-keyed**: `selectedId` (effectiveId) is the `$state`, `selectedIndex` is `$derived` from it (working-copy fallback when the id isn't in the list) ÔÇö all index consumers keep reading a number, but a row-shifting refresh can't re-bind the cursor/diff to a different revision; `loadLog` reconciles identity (stays on the same revision, falls back to `@` only on disappearance). **Server state is loader + op-sync pairs** (the "Op-synced server state" block): every repo-scoped resource (log, workspaces, aliases, remotes, PRs, stale-immutable, oplog/bookmarks panels) is a `createLoader` + `createOpSync` declaration; App holds only `currentOpId` (written by the record-only `onStale` subscription) + the shared gate closure (`opSyncGate`). The EVOLOG is the deliberate exception ÔÇö selection-scoped per-revision data, no op-sync; loaded explicitly at toggle-open, the 50ms selection-follow debounce, and loadLog's post-reconciliation chain (an op-sync run thunk would read the pre-reconciliation selection). ## frontend/src/lib/
